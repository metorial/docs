Workforce is where you decide how access should work after the project already exists. Use it to map who needs access, what they should see, which tools and skills are approved, and where you will review activity.

<Note>
  **What you'll learn:**

  - How to orient around the Workforce dashboard
  - How to decide which accounts, agents, portals, integrations, and skills you need
  - How to choose the first concrete setup path
</Note>

## Open Workforce

Open **Workforce** from the dashboard. This is the control plane for accounts, agents, portals, skills, Magic MCP access, and activity.

![Workforce dashboard](/images/dashboard-audit/workforce-dashboard.png)

## Choose Who Needs Access

Start with the actors that will use tools:

| Actor | Use it for |
| --- | --- |
| Accounts | People, customers, partners, or employees who receive access through Workforce |
| Agents | Non-human actors or linked clients that need durable tool access |

![Accounts page](/images/dashboard-audit/dashboard-accounts.png)

![Agents page](/images/dashboard-audit/dashboard-agents.png)

## Choose The User Surface

Decide whether users should discover access through a portal, connect through Magic MCP, or both.

| Surface | Use it when |
| --- | --- |
| Portal | Users need a branded place to discover approved integrations and skills |
| Magic MCP | Agents or MCP clients need a managed endpoint for approved tool access |
| Skills | Users need reusable workflows built on top of approved integrations |

![Portal admin overview](/images/dashboard-audit/portal-admin-overview.png)

## Pick The First Resources

Keep the first setup small. Choose one or two integrations, one useful skill, and one access group. This makes it easier to test the consumer experience before expanding the portal or adding more clients.

Good first resources are:

- a GitHub or Linear integration
- one skill that solves a real repeated workflow
- one access group for the first user cohort
- one Magic MCP server if agents or MCP clients need direct connection

## Review Activity

After users or agents connect, use Logs to inspect sessions, connections, tool calls, provider runs, errors, alerts, and auth events.

![Logs activity](/images/dashboard-audit/dashboard-logs-activity.png)
