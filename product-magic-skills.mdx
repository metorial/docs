Magic Skills turn repeatable work into reusable assets. Create a skill once, organize it with groups or marketplaces, connect it to integrations, and publish it through Workforce or a portal.

<Note>
  **What you'll learn:**

  - What Magic Skills are for
  - How skills, groups, templates, marketplaces, and settings fit together
  - How skills appear in portals for users
</Note>

## What Skills Are For

Use Magic Skills for work that should be repeatable, shareable, and governed:

- support triage playbooks
- engineering review workflows
- customer handoff steps
- research and reporting tasks
- internal operations checklists

![Magic Skills dashboard](/images/dashboard-audit/workforce-magic-skills.png)

## Create A Skill

Create a skill with a clear name and description. The name should describe the job the user wants done, not the implementation detail behind it.

| Field | Good pattern |
| --- | --- |
| Name | "Summarize Linear Escalations" |
| Description | "Reviews recent Linear issues and drafts a customer-ready escalation summary." |

After a skill exists, add the files, agents, resources, and integrations it needs.

## Connect Integrations

Skills become more useful when they are connected to approved integrations. A customer support skill might use Linear and GitHub; an operations skill might use Google Drive and Slack.

Linked integrations appear on the portal skill card so users can understand what access the workflow uses.

## Organize Skills

| Surface | Use it for |
| --- | --- |
| Skills | Individual reusable workflows |
| Groups | Sets of related skills, such as a team playbook |
| Templates | Starting points for new skills |
| Marketplaces | Curated skill catalogs for users |
| Settings | Default execution policy |

![Magic Skills groups](/images/dashboard-audit/workforce-skill-groups.png)

## Publish Skills Through Portals

Portals can publish individual skills, templates, or groups. Add a skill resource to the portal, choose the skill, and allow the right access groups.

![Portal skill listings](/images/dashboard-audit/portal-admin-skills.png)

Users then see the skill inside the portal with its connected integrations.

![Consumer portal skills](/images/dashboard-audit/portal-consumer-skills.png)

## Govern Skill Execution

Settings define the default policy for skill files and execution behavior.

![Magic Skills settings](/images/dashboard-audit/workforce-skill-settings.png)

| Setting | What it controls |
| --- | --- |
| Allow scripts | Whether skills can include and execute script files |
| Allow non-standard directories | Whether skill files can live outside the standard skill directory structure |
| Allow all file extensions | Whether skill uploads are limited to an approved extension list |

## Magic Skills Versus Provider Skills

Magic Skills are reusable Workforce assets. Provider skills are catalog summaries on provider pages.

| Concept | Meaning |
| --- | --- |
| Magic Skill | A reusable workflow created and governed in Workforce |
| Magic Skill Group | A collection of related Magic Skills |
| Skill Marketplace | A curated place to publish selected skills and plugins |
| Provider Skill | A provider-page summary that describes what a catalog provider can do |
