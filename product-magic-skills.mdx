Magic Skills are part of the Workforce product. They let teams package reusable agent workflows and make them available through controlled marketplaces, templates, groups, and execution policies.

<Note>
  **What you'll learn:**

  - What the Magic Skills section contains after starting a Workforce trial
  - How skills, marketplaces, templates, and groups fit together
  - Which settings control skill execution policy
</Note>

## Dashboard Surface

After Workforce is active, **Magic Skills** appears in the Workforce sidebar with five tabs:

- **Skills**
- **Marketplaces**
- **Templates**
- **Groups**
- **Settings**

![Magic Skills dashboard](/images/dashboard-audit/workforce-magic-skills.png)

## Skills

Skills are reusable workflows that can combine integrations, documents, custom logic, and resources. The Skills page lets you search, filter, and create skills.

The create flow starts with:

| Field | Purpose |
| --- | --- |
| Name | Human-readable skill name |
| Description | Short explanation of what the skill does |

## Marketplaces

Marketplaces let you publish selected plugins and skills to users. Use them when different teams or consumers should see curated skill catalogs instead of every available skill.

## Templates

Templates are reusable starting points for new skills. The default trial view includes a **Plain** template for creating a skill with no preconfigured items.

## Groups

Groups organize related skills and let you manage them as a set.

![Magic Skills groups](/images/dashboard-audit/workforce-skill-groups.png)

Use groups when you want to keep related workflows together, such as a support playbook, engineering review workflow, or onboarding collection.

## Settings

Settings define the default execution policy used by skills in the instance.

![Magic Skills settings](/images/dashboard-audit/workforce-skill-settings.png)

The current settings are:

| Setting | What it controls |
| --- | --- |
| Allow scripts | Whether skills can include and execute script files |
| Allow non-standard directories | Whether skill files can live outside the standard skill directory structure |
| Allow all file extensions | Whether skill uploads are limited to an approved extension list |

## Magic Skills Versus Provider Skills

Magic Skills are reusable Workforce assets. Provider skills are catalog summaries on provider pages.

| Concept | Meaning |
| --- | --- |
| Magic Skill | A reusable workflow created and governed in the Workforce product |
| Magic Skill Group | A collection of related Magic Skills |
| Skill Marketplace | A curated place to publish selected skills and plugins to users |
| Provider Skill | A provider-page summary that describes what a catalog provider can do |
