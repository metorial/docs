title: "Preview Portal Access"
description: "Preview the consumer portal and confirm users can see the right integrations and skills."
---

Test the portal from the consumer point of view before sharing it. This catches missing access groups, hidden resources, and confusing portal copy early.

<Note>
  **What you'll learn:**

  - How users authenticate into a portal
  - How the portal home page presents approved resources
  - How to verify integrations and skills from the consumer view

  **Before you start:**

  - Complete [Workforce Concepts](/metorial-101-introduction)
  - Complete [Create A Portal](/metorial-101-deploying)
</Note>

## Open The Portal

Open the portal URL and complete the configured authentication flow. If email verification or SSO is enabled, test the same path your users will use.

![Portal consumer login](/images/dashboard-audit/portal-consumer-email-verification.png)

## Review The Home Page

The portal home page should make the approved resources obvious: Magic MCP connection details, highlighted resources, available integrations, and shared skills.

![Consumer portal home](/images/dashboard-audit/portal-consumer-home.png)

## Check Integrations

Open the integrations view and confirm the right tools are visible. Resources that require user credentials should make the connection state clear.

![Consumer portal integrations](/images/dashboard-audit/portal-consumer-integrations.png)

## Check Skills

Open the skills view and confirm the published skills appear with the integrations they use. If a skill is missing, return to the portal admin resource settings and verify the access group is allowed.

![Consumer portal skills](/images/dashboard-audit/portal-consumer-skills.png)

## Test Provider Behavior

Use Explorer when you need to inspect the underlying provider tools before exposing them broadly.

![Explorer](/images/dashboard-audit/dashboard-explorer.png)

## What's Next?

Once the portal looks right, review activity and operational logs.
