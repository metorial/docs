The Metorial dashboard is where you configure governed access for people and agents. Start with Workforce, then add portals, integrations, skills, and Magic MCP access as your users need them.

<Note>
  **What you'll learn:**

  - Where the main Workforce product surfaces live
  - How Portals, Skills, Integrations, and Magic MCP connect
  - Where to inspect activity, logs, and workspace settings
</Note>

## Onboarding

New accounts start on the welcome flow. Create an organization, choose a data region, and create your first project.

![Create an organization and choose a data region](/images/dashboard-audit/onboarding-create-organization-eu.png)

The region choice is made during organization setup. In this walkthrough, the organization uses the **European Union** region.

![Create a project](/images/dashboard-audit/onboarding-create-project-filled.png)

## Home

The project home page gives you the current organization, project, and environment context. It also shows setup shortcuts and popular catalog providers such as GitHub and Linear.

![Dashboard home](/images/dashboard-audit/dashboard-home-after-onboarding.png)

## Workforce

Workforce is the product home for access management. Use it to review accounts, portals, activity, Magic Skills, and access surfaces connected to your project.

![Workforce dashboard](/images/dashboard-audit/workforce-dashboard.png)

## Portals

Portals let you publish a branded catalog of approved integrations and skills. Admins configure access groups, resource listings, authentication, and highlights; users open the portal to connect and discover what they can use.

![Portal admin overview](/images/dashboard-audit/portal-admin-overview.png)

![Consumer portal home](/images/dashboard-audit/portal-consumer-home.png)

## Magic Skills

Magic Skills are reusable workflows your team can create, organize, and publish. Use skills for repeatable work such as support playbooks, engineering review steps, onboarding flows, or customer-facing automations.

![Magic Skills dashboard](/images/dashboard-audit/workforce-magic-skills.png)

## Integrations And Providers

Use **Providers** to browse catalog providers and **Integrations** to configure reusable access. Provider pages include tools, capabilities, versions, and setup actions for integrations or Magic MCP servers.

![Provider catalog](/images/dashboard-audit/dashboard-providers-catalog.png)

![GitHub integration setup](/images/dashboard-audit/provider-github-create-integration-github-com-selected.png)

## Magic MCP

Magic MCP servers turn configured providers into connectable MCP endpoints for agents and clients. Use groups and tokens to control which clients can connect to which servers.

![Magic MCP servers](/images/dashboard-audit/dashboard-magic-mcp-servers.png)

## Activity And Logs

Use Logs to inspect sessions, connections, tool calls, provider runs, errors, alerts, Protoguard events, and provider-auth events. This is where operators confirm that access is being used as expected.

![Logs activity](/images/dashboard-audit/dashboard-logs-activity.png)

## Settings

Organization settings include general settings, projects, members, and invites. Project settings live under the organization project route.

![Organization settings](/images/dashboard-audit/dashboard-organization-settings.png)
